# Mini Challenge 4 - PostArchivedListView and PostDraftListView
Create views to display the posts that have an archived status and draft status.

STR:
1. Create two more views that inherits from ListView
2. Create the urls to handle the views
3. Create the templates for those views.

Criteria:
- Drafts should only return *all* posts from the *logged in* user.
- Archived should return *all* posts with that status and the logged in user should be able to modify their own posts.
- Add the options to the navbar and make sure that only logged in users can view those options.
- Make sure to address the vulnerabilities if a normal user tries to modify the address url.
- For the DraftPostListView use the get_context_data method to filter the data.
